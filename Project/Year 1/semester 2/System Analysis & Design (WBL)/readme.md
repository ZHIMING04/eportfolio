## 📘 Course Overview

**System Analysis and Design (SECP2613)** introduced the practical process of analyzing, planning, designing, and proposing information systems for real organizational problems.

## 🛠️ Coursework

### 🏢 KADA Digital Management System

For the group project, my team proposed a digital management system for **Koperasi Kakitangan KADA Kelantan Bhd**.

Based on the interview and requirement gathering process, we found that KADA was still relying heavily on **physical forms and manual document handling** for member registration, loan applications, savings records, and report generation.

The existing process caused several issues:

* Manual and time-consuming data entry
* Physical form storage
* Higher risk of document loss
* Limited accessibility for members
* Difficulty generating reports
* Members needing to contact the office to check savings or account details

Our proposed system aimed to digitize the workflow and improve operational efficiency.

## 🎯 Proposed System Scope

The proposed KADA system focused on five core modules:

| Module            | Purpose                                                     |
| ----------------- | ----------------------------------------------------------- |
| User Registration | Allow new users to apply for cooperative membership online  |
| User Login        | Provide authentication for members                          |
| Loan Application  | Allow members to apply for loans digitally                  |
| Money Savings     | Track monthly salary deduction and savings records          |
| Reporting         | Generate reports for members and management decision-making |

The system involved four main user groups:

* New users
* Members
* Clerks
* Board of Directors

## 🧠 Reflection

This project helped me understand how real systems are planned before development begins. Instead of jumping directly into coding, I learned the importance of understanding the client’s current process, identifying pain points, gathering requirements, and modelling the proposed solution clearly.

The KADA project showed me that digital transformation is not only about building a website. It also involves understanding users, improving workflows, reducing manual workload, securing data, and supporting better decision-making.

My main takeaway from this subject is that a good system starts with strong analysis. Clear requirements, proper planning, and well-designed workflows are just as important as implementation.
