## 📘 Course Overview

**Computer Organization and Architecture (SECR1033)** introduced how computer systems work at the hardware and instruction level.

Through this course, I learned about computer components, data representation, arithmetic operations, instruction set architecture, CPU execution, pipelining, memory systems, input/output, storage systems, and performance measurement.

This subject helped me understand how software instructions interact with hardware components inside a computer.

---

## 🛠️ Coursework

### 🚀 CPU Benchmarking Project

For the group project, we analyzed and compared the performance of four laptops using several benchmarking tools:

| Tool                      | Purpose                                    |
| ------------------------- | ------------------------------------------ |
| CPU-Z                     | Hardware specification and CPU details     |
| Geekbench                 | Single-core and multi-core CPU performance |
| Windows Task Manager      | CPU, memory, and system load monitoring    |
| MiniTool Partition Wizard | Disk read/write performance testing        |

The project compared laptops based on CPU cores, threads, RAM, cache, storage performance, CPU load, and execution time.

We also wrote and executed an **assembly language CPU stress-test program** to measure CPU elapsed time across different loop counts.

---

### ⚙️ Assembly Language Labs

The lab exercises focused on low-level programming using **MASM, Visual Studio, and Irvine32 library**.

Key lab activities included:

* Register inspection using `DumpRegs`
* Arithmetic operations using assembly instructions
* Data movement using `MOV`
* Register swapping using `XCHG`
* Multiplication and division using `MUL` and `DIV`
* Working with hexadecimal values
* Using loops and arrays
* Building menu-based assembly programs
* Applying comparison and conditional jump instructions

## 🧠 Reflection

This course helped me understand computers beyond the surface level. Instead of only using software, I learned how instructions are executed by the CPU, how registers store temporary values, and how hardware specifications affect real performance.

The benchmarking project was especially useful because it showed me that computer performance is not determined by one factor alone. CPU cores, threads, cache, RAM, storage speed, background processes, and system load all affect the final result.

The assembly language labs were challenging but valuable. They trained me to think closer to the machine level and helped me understand how high-level programming is supported by low-level instructions.

Overall, this subject strengthened my foundation in computer systems, hardware performance, and low-level programming.
