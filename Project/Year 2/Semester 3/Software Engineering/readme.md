## 📘 Course Overview

**Software Engineering** introduced the engineering approach used to develop high-quality software systems.

Through this course, I learned how to analyze software problems, choose suitable software process models, define requirements, design system architecture, create UML models, and plan software testing.

The course helped me understand that building software is not only about coding, but also about planning, documentation, teamwork, requirements, design, testing, and continuous improvement.

---

## 🛠️ Coursework

### 🚀 KADA eServe System

For the main project, my team **TechAway** proposed **KADA eServe**, a digital system for **Lembaga Kemajuan Pertanian Kemubu (KADA)**.

The system aimed to replace KADA’s paper-based processes with a more efficient digital workflow for:

* Member registration
* Loan application
* Savings management
* Report generation
* Member financial statement access
* Administrative approval workflow

The project was built around real organizational problems, especially KADA’s reliance on physical forms, manual document handling, slow processing, and limited reporting features.

## 🧠 Reflection

This course helped me understand the importance of structured software development.

Through the **KADA eServe** project, I learned that a successful system requires clear requirements, proper planning, strong documentation, and continuous communication with stakeholders. Choosing Agile also showed me how software can be improved through iteration, feedback, and incremental delivery.

Overall, Software Engineering strengthened my ability to think like a system builder, not only as a programmer.
