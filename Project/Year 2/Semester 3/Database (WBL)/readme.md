## 📘 Course Overview

**Database (SECP2523)** introduced the fundamentals of database systems, relational database design, and SQL-based data manipulation.

Through this course, I learned how to design and develop database systems using **ERD, relational schema, normalization, SQL DDL/DML, joins, stored procedures, and MySQL**.

This subject helped me understand how data should be structured, stored, queried, and connected to real system functionality.

---

## 🛠️ Coursework

### 🧪 Lab Work

The lab activities focused on practical database creation and SQL querying.

Key work included:

* Creating database tables
* Defining primary keys and foreign keys
* Inserting and managing records
* Writing SQL queries
* Using joins and aggregate functions
* Practicing database operations with sample business cases

One lab involved creating a **Tennis Logos database** with supplier and item tables. Another lab involved SQL queries for an online bookstore database, including retrieving books, customers, orders, revenue, popular genres, and customer spending.

## 🚀 Project: KADA-eServe

For the main project, my team developed **KADA-eServe**, a database-driven system designed to improve KADA’s administrative workflow.

My module focused on the **Administrative Module**, which supports:

* Member registration approval
* Loan application approval
* Transaction management
* Transaction history
* Statistical dashboard reporting
* SQL-based administrative data retrieval

The project involved designing and implementing database structures for entities such as:

| Entity              | Purpose                                     |
| ------------------- | ------------------------------------------- |
| Members             | Store approved member information           |
| Admin               | Manage administrative users                 |
| Loans               | Store loan application records              |
| Transactions        | Track savings and loan-related transactions |
| Loan Types          | Classify loan categories                    |
| Banks               | Store member banking information            |
| Guarantors          | Store guarantor details                     |
| Family / Staff Info | Support member profile information          |

---

## 🧩 Database Design Work

The project included:

* Conceptual ERD
* Logical ERD
* Data dictionary
* Relational database schema
* Normalization up to 3NF / BCNF
* SQL DDL table creation
* SQL DML queries
* JOIN queries for dashboards and reports
* Interface implementation linked with SQL queries

Example implemented queries included:

* Retrieve pending loan applications with member details
* Approve loan applications
* Approve member registration
* Insert loan transaction records
* View member transaction history
* Generate loan approval statistics by loan status

## 🧠 Reflection

This course helped me understand that a strong system depends heavily on a strong database design.

Through **KADA-eServe**, I learned that poor database planning can create many problems during implementation, especially when relationships, attributes, and table structures are not clearly defined early. The project taught me the importance of requirement analysis, normalization, and consistent schema design before development begins.

I also gained hands-on experience writing SQL queries that directly support real system features such as loan approval, member approval, transaction tracking, and dashboard reporting.

Overall, Database strengthened my foundation in data modelling, SQL, and backend system thinking, which are important skills for my future direction in data engineering.
