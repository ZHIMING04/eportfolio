## 📘 Course Overview

**Network Communications (SECR1213)** introduced the fundamentals of computer networking using the **TCP/IP protocol stack**.

Through this course, I learned how data moves across networks through different layers, including the **application layer, transport layer, network layer, link layer, and wireless networks**.

This subject helped me understand how real network communication works behind everyday applications such as web browsing, email, DNS lookup, routing, and wireless connectivity.

---

## 🛠️ Coursework & Labs

### 🔎 Wireshark Packet Analysis

Used **Wireshark** to analyze real packet traces for:

* HTTP GET / response messages
* DNS queries and responses
* IP addresses, ports, TTL, and status codes
* Application-layer protocol behaviour

### 📦 TCP & UDP Simulation

Used **Cisco Packet Tracer** to compare TCP and UDP communication.

Key observations:

* TCP uses sequence numbers, acknowledgements, and reliable sessions
* UDP is faster but does not guarantee delivery
* DNS uses UDP port 53
* HTTP, FTP, SMTP, and POP3 rely on TCP-based communication

### 🛣️ Routing Protocol Lab

Configured routing using **RIP** in Packet Tracer.

The lab included:

* IPv4 subnetting
* Router interface configuration
* Routing table analysis
* RIP routing configuration
* End-to-end connectivity testing between PCs

### 🔁 ARP & Switch Table Lab

Explored how devices discover MAC addresses and how switches learn forwarding information.

The lab covered:

* ARP table checking
* MAC address table observation
* Ping-based traffic generation
* Switch forwarding behaviour
* Wireless LAN configuration activity

## 🚀 Project: Faculty Network Design

For the group project, we acted as network consultants to design a network for **Faculty of Computing Block N28B**.

The project required us to plan a scalable and reliable network for:

* 4 computer labs
* 1 Cisco networking lab
* 1 embedded / IoT lab
* 1 hybrid classroom
* 1 video conferencing room
* Student lounge with Wi-Fi access

The project involved:

* Requirement analysis
* Floor plan planning
* Network device selection
* LAN and WAN connection planning
* Cable type and cable length estimation
* IPv4 subnetting
* Budget consideration
* Professional network documentation

## 🧠 Reflection

This course helped me understand networking from both theory and practical perspectives.

The labs showed me how protocols such as **HTTP, DNS, TCP, UDP, ARP, and RIP** actually behave in a network. Instead of only memorizing definitions, I was able to observe packets, inspect headers, configure routers, and test connectivity.

The network design project also taught me that building a network is not only about connecting devices. It requires planning around user needs, scalability, budget, physical cabling, IP addressing, performance, and security.

Overall, Network Communications strengthened my foundation in infrastructure thinking, which is important for cloud, DevOps, and data engineering work.
