## 📘 Course Overview

**Operating Systems** introduced how an operating system manages computer hardware, processes, memory, files, storage, and I/O resources.

Through this course, I learned about **process management, threads, CPU scheduling, synchronization, deadlock, memory allocation, virtual memory, file systems, disk scheduling, and Linux system operations**.

This subject helped me understand how software interacts with system-level resources behind the scenes.

---

## 🛠️ Coursework & Labs

### 🐧 Lab 1: Basic Linux Commands & C Programming

Practiced basic Linux operations and command-line tools, including:

* Directory navigation
* File creation, copying, renaming, and deletion
* Text file manipulation using `echo`, `cat`, and `touch`
* Writing, compiling, and running C programs using `gcc`

---

### ⚙️ Lab 2: Linux Process Monitoring & Management

Practiced process-related commands in Linux, including:

* Listing running processes using `ps`
* Filtering process information with `grep`
* Viewing process hierarchy using `ps --forest` and `pstree`
* Sorting processes by memory usage
* Changing process priority using `renice`
* Terminating processes using `killall`
* Demonstrating multiprocessing using `fork()` and `wait()`

---

### 📁 Lab 3: File Management & Bash Scripting

Practiced Linux file system management using bash scripting.

Key tasks included:

* Creating directories and files using scripts
* Generating tree structures
* Filtering files by extension
* Counting files dynamically
* Managing file permissions using `chmod`
* Understanding access control for owner, group, and public users

---

## 🚀 Group Project: Non-Preemptive Scheduling Simulation

For the group project, my team **Error 404** simulated two non-preemptive CPU scheduling algorithms using **C++**:

| Algorithm           | Scenario                  | Description                                                       |
| ------------------- | ------------------------- | ----------------------------------------------------------------- |
| Shortest Job First  | Print Queue System        | Prioritizes print jobs with fewer pages                           |
| Priority Scheduling | Airport Runway Scheduling | Prioritizes emergency flights over commercial and private flights |

My contribution focused on **simulation coding, testing, and debugging**.

## 🧠 Reflection

This course helped me understand what happens behind normal program execution.

Through the labs, I became more comfortable using Linux commands, managing files, monitoring processes, and writing small C programs that interact with operating system concepts.

The scheduling project was especially useful because it connected OS theory with real-world scenarios. Simulating **SJF** using a print queue and **Priority Scheduling** using airport runway management helped me understand how scheduling decisions affect waiting time, fairness, efficiency, and starvation risk.

Overall, Operating Systems strengthened my system-level thinking and gave me a stronger foundation for backend development, cloud infrastructure, DevOps, and data engineering work.
