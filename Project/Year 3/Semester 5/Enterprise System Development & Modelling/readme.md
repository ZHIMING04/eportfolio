## 📘 Course Overview

**Enterprise System Design and Modeling** introduced how enterprise information systems are planned, integrated, and aligned with organizational goals.

Through this course, I learned about **Enterprise Resource Planning (ERP), Enterprise Architecture (EA), system integration, TOGAF, Zachman Framework, enterprise system lifecycle, value chain modeling, data governance, SAP technologies, and work-based learning case studies**.

This subject helped me understand how large organizations design integrated systems that connect business processes, data, applications, and technology.

---

## 🛠️ Coursework

### 🏫 Main Project: UTHM Inventory System

For the main project, I worked on an **Integrated Inventory Management System for UTHM**.

The project focused on solving problems caused by fragmented systems, manual asset tracking, information silos, and limited real-time visibility between departments.

The proposed system aimed to support UTHM’s digital transformation by introducing:

* Centralized inventory data
* Enterprise Architecture design using TOGAF
* Three-tier system architecture
* Single Sign-On access
* Real-time asset tracking
* Repair and defect reporting module
* QR-based asset identification
* SAP BTP-based implementation concept
* Dashboard and reporting features

---

## 🧩 My Key Contribution

My main focus was on the **system design, enterprise architecture, ERD, class diagrams, and repair subsystem modeling**.

The repair subsystem was designed to manage the lifecycle of damaged assets:

**Available → Under Repair → Available / Disposed**

Key design artifacts included:

| Artifact             | Purpose                                                                          |
| -------------------- | -------------------------------------------------------------------------------- |
| Use Case Diagram     | Shows interaction between staff, technicians, and repair system                  |
| Use Case Description | Defines the defect reporting workflow                                            |
| Activity Diagram     | Models the full repair process and auto-blocking logic                           |
| Sequence Diagram     | Shows how report submission updates asset status                                 |
| ERD                  | Defines repair logs, asset records, and user relationships                       |
| Interface Design     | Includes staff portal, reporting page, asset dashboard, and technician dashboard |

## 🧠 Reflection

This course changed how I view system development.

Before this, I mainly saw systems as applications with features. Through the UTHM case study, I learned that enterprise systems are much bigger than that. They involve business strategy, organizational structure, data governance, infrastructure, integration, security, and long-term scalability.

The most valuable part was designing the **repair subsystem**, especially the auto-blocking workflow that prevents damaged assets from being borrowed. It showed me how good system architecture can enforce business rules automatically and reduce operational risk.

Exploring SAP BTP and SAP Build Apps also gave me early exposure to enterprise-grade platforms, low-code development, and how Generative AI can speed up application prototyping.

Overall, this subject strengthened my ability to think like a **solutions architect**, not only a developer.
