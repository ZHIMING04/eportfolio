## 📘 Course Overview

**Database Programming** deepened my understanding of relational and non-relational database systems.

Through this course, I learned about **advanced SQL operations, stored procedures, PL/SQL, indexing, transactions, concurrency control, isolation levels, locking, MongoDB, NoSQL data models, and database application development**.

This subject helped me move beyond basic database design into writing stronger, safer, and more scalable database logic.

---

## 🛠️ Coursework

### 🔐 Assignment: Transaction Management

For Assignment 2, I worked on database transaction management and concurrency control problems.

The assignment involved analyzing multiple transaction schedules and applying database control techniques to ensure correctness.

Key tasks included:

* Identifying serial and non-serial schedules
* Drawing precedence graphs
* Checking conflict serializability
* Applying shared and exclusive locks
* Applying Two-Phase Locking protocol
* Identifying concurrency problems such as deadlock, lost update, starvation, and uncommitted dependency

---

## 🧩 Key Concepts Practiced

| Concept               | Description                                                                         |
| --------------------- | ----------------------------------------------------------------------------------- |
| Serializability       | Checking whether concurrent transactions produce a correct equivalent serial result |
| Precedence Graph      | Visualizing conflicts between transactions                                          |
| Locking               | Using shared and exclusive locks to control concurrent access                       |
| Two-Phase Locking     | Ensuring transactions follow growing and shrinking phases                           |
| Deadlock              | Understanding situations where transactions wait indefinitely                       |
| Lost Update           | Identifying when one transaction overwrites another transaction’s result            |
| Transaction Isolation | Understanding how transaction behavior affects consistency                          |

## 🧠 Reflection

This course helped me understand that database systems are not only about storing and retrieving data. They also need to handle multiple users, simultaneous transactions, consistency, and reliability.

The transaction management assignment was especially useful because it showed how easily problems such as **lost updates, deadlocks, starvation, and uncommitted dependencies** can happen when transactions run concurrently without proper control.

Overall, Database Programming strengthened my backend and data engineering foundation by helping me think more carefully about data consistency, transaction safety, and scalable database system design.
